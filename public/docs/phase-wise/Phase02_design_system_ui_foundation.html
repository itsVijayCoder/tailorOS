Phase 02 · UI Foundation

UI foundation and premium experience goal

Phase 02 establishes the premium TailorOS interface before feature screens multiply. The product should feel like a polished shop operating desk: fast search, elegant cards, calm light-golden surfaces, visible order progress, and tasteful micro-interactions that confirm action without slowing staff.

UI decision. Build the design system as owned shadcn/ui components, Tailwind tokens, and Motion wrappers. Use TanStack Query for server state, Zustand for local UI state, React Hook Form + Zod for forms, and keep animations transform/opacity-based with reduced-motion support.

Component-level structure

Components should be split by design-system primitives, reusable business UI, and feature-only components. This prevents the common failure where every feature creates a new button/card/status chip.

Component rules

  • Primitives never import TailorOS business types.
  • Business components can accept domain DTOs but should not fetch data directly.
  • Feature components may call hooks, but mutations belong in feature hooks, not individual buttons.
  • Every interactive component has loading, disabled, success, error, and keyboard states.

Phase 02 · UI Foundation

Micro-interaction system

Motion should communicate state. TailorOS staff will use the app during busy shop hours, so effects must be elegant, brief, and never block actions.

InteractionMotionDurationReason
Button tapScale to 0.98 and return.90–120msImmediate tactile feedback.
Command palette openOpacity + scale from 0.98.160–200msPremium search entry.
Drawer/modalTransform slide + fade overlay.180–240msClear spatial model.
Order status changedNew timeline node fades and lifts 6px.180msConfirms workflow step.
WhatsApp statusBadge text crossfades; no row re-layout.140msLive feel without distraction.
Page transitionContent opacity and y transform only; shell stays fixed.180–260msFast perceived navigation.
'use client';
import { motion, useReducedMotion } from 'motion/react';

export function AnimatedPage({ children }: { children: React.ReactNode }) {
  const reduce = useReducedMotion();
  return (
    <motion.div
      initial={reduce ? false : { opacity: 0, y: 8 }}
      animate={reduce ? undefined : { opacity: 1, y: 0 }}
      exit={reduce ? undefined : { opacity: 0, y: -4 }}
      transition={{ duration: 0.18, ease: [0.2, 0.8, 0.2, 1] }}
    >
      {children}
    </motion.div>
  );
}

Phase 02 · UI Foundation

Tests, accessibility, and exit checklist

Design-system tests

  • Visual regression for buttons, inputs, cards, status chips, timeline, receipt, inbox bubble.
  • Accessibility checks for keyboard navigation, focus rings, labels, contrast, and dialog trapping.
  • Reduced-motion test to verify non-essential animations are disabled.
  • Mobile viewport snapshots: 360px, 390px, 768px, 1024px, 1440px.
  • Performance test: command palette opens quickly with 100+ cached search results.

Exit checklist

  • Light golden theme tokens are documented and available in CSS variables.
  • Every major UI state has loading, empty, error, success, blocked, and permission-denied variants.
  • Micro-interactions use transform/opacity and respect reduced motion.
  • Components are organized into primitives, layout, feedback, data-display, and business components.
  • Design-system demo file is available for stakeholder review.
