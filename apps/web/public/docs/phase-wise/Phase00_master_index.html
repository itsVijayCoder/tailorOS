Recommended build sequence

  1. Phase 01–02 first: engineering foundation and UI design system.
  2. Phase 03–04 next: tenant provisioning and reliable domain model.
  3. Phase 05 pilot: core TailorOS modules usable by one shop.
  4. Phase 06 integrate WhatsApp after clean order/payment/status data exists.
  5. Phase 07–09 harden speed, security, testing, observability, and release.
  6. Phase 10 prepares scale and future architecture.

Master · Index

Quality principles used

  • SOLID boundaries: route, service, repository, adapter, and domain layers are separated.
  • DRY where it matters: schemas, state machines, ID rules, permissions, and UI tokens are shared.
  • Tenant isolation: control plane plus per-tenant D1 databases.
  • Reliability: outbox pattern, queues, DLQ, idempotency, and audit logs.
  • Premium UX: light golden theme, fast command search, reduced-motion-friendly micro-interactions.
