Phase 09 · QA & Release

Testing, observability, and release objective

Phase 09 creates the verification system that makes TailorOS safe to ship: test pyramid, Worker runtime tests, webhook fixtures, Playwright journeys, logs, metrics, alerts, CI/CD gates, staging, release checklist, and support runbooks.

Senior engineering rule. If a production bug cannot be traced to tenant, request, user, order, message, and deployment version, the observability system is incomplete.

Phase 09 · QA & Release

CI/CD and release gates

Pull request → lint → typecheck → unit tests → schema/contract tests → Worker integration tests → build web + Workers → preview deploy → Playwright smoke → staging migration → manual release approval → production deploy → post-deploy smoke → monitor error/latency/DLQ
  • Separate staging and production Cloudflare resources.
  • Never run production migrations from developer laptop without audit.
  • Tenant migrations use fan-out queue and per-tenant status.
  • Deploy API changes before frontend changes when adding backward-compatible fields.
  • Use feature flags for risky new modules.

Phase 09 · QA & Release

Runbooks and support playbooks

  • Tenant provisioning stuck in db_migrating.
  • Tenant DB migration partially failed.
  • Search shows wrong/missing customer.
  • WhatsApp webhook verification fails.
  • WhatsApp messages in DLQ.
  • Receipt public link not opening.
  • D1 database near storage limit.
  • Support access request and review process.
  • Rollback after bad frontend deployment.
  • Emergency disable WhatsApp sends for one tenant.

Phase 09 · QA & Release

Release acceptance checklist

Go-live checklist for first real shop

  • Seed shop settings, garment templates, receipt branding, and roles.
  • Import at least 50 historical customers and 20 active orders.
  • Train owner/counter staff on search, family selection, measurement versions, order creation, payment correction, WhatsApp opt-in.
  • Run parallel notebook comparison for two weeks.
  • Monitor order entry time, search success, missing promised dates, payment corrections, WhatsApp failures.
  • Conduct weekly pilot review and prioritize issues before onboarding more shops.
